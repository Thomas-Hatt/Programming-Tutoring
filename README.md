## üìñ How Content Pages Are Structured
Each individual program page (like the one you mentioned for Program 5) is structured to mirror the assignment's requirements, ensuring you address every component of the task:

### Task
A brief, real-world scenario and the primary objective of the program.

### Input
Details on what the user must enter, including specific prompts, data types, and required letter-based function selections (e.g., 'C' for Calculate, 'D' for Display, 'X' for Exit).

Process & Functions
This section is critical and outlines the exact structure required for the Python program, including required function names, their arguments, and their return values.

### Example Function Requirements:

DisplayMenu(): Accepts zero arguments, returns nothing. (e.g., displays current prices).

CalcTotal(): Accepts zero arguments, returns a single value (e.g., the cost of the selected item).

CalcTax(subtotal): Accepts one argument, returns a single value (e.g., calculated sales tax).

Output Scenarios
Clear examples showing exactly what the program should display for different user actions (e.g., selecting 'C' to calculate, 'D' to display the menu, or 'X' to exit).
